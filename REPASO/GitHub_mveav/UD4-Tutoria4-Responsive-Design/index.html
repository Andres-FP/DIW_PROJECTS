Definir la codificación de caracteres

Al igual que con los archivos HTML, es una buena idea en cada documento CSS definir la codificación de caracteres utilizado en el archivo. En CSS, esto se consigue con la siguiente regla @charset

@charset "codificación";

donde encoding define la codificación de caracteres utilizada en el archivo. Añada la regla @charset al archivo de hoja de estilo base.css, especificando que se utilizará el conjunto de caracteres UTF-8 en el código CSS

Importar hojas de estilos

La regla @charset es un ejemplo de una at-rule CSS, que es una regla utilizada para enviar directivas al navegador indicando cómo debe interpretarse y analizarse el contenido del archivo CSS. Otra at-rule es la siguiente @import utilizada para importar el contenido de un archivo de hoja de estilo

@import url(url);

donde url es la URL de un archivo de hoja de estilo externo.

La regla @import se utiliza para combinar reglas de estilo de varias hojas de estilo en un único archivo.

Por ejemplo, una tienda online puede tener una hoja de estilo llamada basic.css que contenga todos los estilos básicos utilizados en cada página web y otra hoja de estilo especifica llamada tienda.css para la parte de la tienda oline.

La declaración @import debe ir siempre antes de cualquier otra regla de estilo en la hoja de estilo.

De forma que el siguiente código importa la hoja de estilos base en la ce tienda.css:

@import url(base.css);

Si tenemos distintas hojas de estilos a importar estas se ponen una debajo de la otra. El uso de múltiples reglas @import en un archivo CSS tiene el mismo impacto que añadir múltiples elementos de enlace al archivo HTML. Una de las ventajas de la regla @import es que simplifica su código HTML al colocar la decisión sobre qué hojas de estilo incluir y excluir en el archivo CSS en lugar de en el archivo HTML.

En nuestro caso vamos a importar normalize.css en la hoja de estilos base.css

Las pseudo-clases y los pseudo-elementos

La diferencia entre pseudo-clases y pseudo-elementos es sutil. En general, los pseudo-elementos suelen hacer referencia a determinadas partes de un elemento, mientras que las pseudo-clases suelen hacer referencia al estado del elemento.

En resumen podemos decir que:

  1. Las pseudo-clases y los pseudo-elementos se utilizan para agregar efectos especiales al selector.
  2. La diferencia esencial entre las pseudo-clases y los pseudo-elementos es si se deben crear nuevos elementos de forma abstracta en dicho caso se usan los pseudo-elementos.
  3. Las pseudo-clases se pueden superponer siempre que no sean mutuamente excluyentes.
  4. El pseudo-elemento solo puede aparecer una vez en un selector y solo puede aparecer al final.
  5. La prioridad de pseudo-clase y pseudo-elemento es la misma que la prioridad de clase y etiqueta respectivamente.
  6. Las pseudo-clases van precedidas de ":" y los pseudo-elementos por "::" aunque también admiten un solo ":" e incluso hay navegadores con IE que no soportan los "::".
  7. img, input y otras etiquetas individuales no tienen pseudoelementos antes y después, porque las etiquetas individuales en sí mismas no pueden tener elementos secundarios. Si queremos emplear pseudo-elemtos antes y despues los metemos detro de un elemento tipo contenedor como un div, aside, article,...

Ejemplo pseudo-elementos

Vamos a emplear la pseudo-case nt-child para poder colorear el fondo de las filas pares e impares de forma distinta.

Con la pseudo-clase nth-child(odd) hacemos referencia a las filas impares, mientras que con nth-child(even) hacemos referencia a las pares.

tbody tr:nth-child(odd){
background-color: #f8e2dca8;
}

También podemos usar

tbody tr:nth-child(2n+1){
background-color: #f8e2dca8;
}

Otro método sería crear una clase para las filas impares o bien un id para cada fila a colorear el fondo.

#fila1, #fila3{
background-color: #f8e2dca8;
}

O

.Impares{
background-color: #f8e2dca8;
}
