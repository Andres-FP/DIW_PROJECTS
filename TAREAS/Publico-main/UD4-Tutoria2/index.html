UD4 - Estilo de texto, pseudo-clases y pseudo-elementos

Definir la codificación de caracteres

Al igual que con los archivos HTML, es una buena idea en cada documento CSS definir la codificación de caracteres utilizado en el archivo. En CSS, esto se consigue con la siguiente regla @charset

@charset "codificación";

donde encoding define la codificación de caracteres utilizada en el archivo. Añada la regla @charset al archivo de hoja de estilo base.css, especificando que se utilizará el conjunto de caracteres UTF-8 en el código CSS

Importar hojas de estilos

La regla @charset es un ejemplo de una at-rule CSS, que es una regla utilizada para enviar directivas al navegador indicando cómo debe interpretarse y analizarse el contenido del archivo CSS. Otra at-rule es la siguiente @import utilizada para importar el contenido de un archivo de hoja de estilo

@import url(url);

donde url es la URL de un archivo de hoja de estilo externo.

La regla @import se utiliza para combinar reglas de estilo de varias hojas de estilo en un único archivo.

Por ejemplo, una tienda online puede tener una hoja de estilo llamada basic.css que contenga todos los estilos básicos utilizados en cada página web y otra hoja de estilo especifica llamada tienda.css para la parte de la tienda oline.

La declaración @import debe ir siempre antes de cualquier otra regla de estilo en la hoja de estilo.

De forma que el siguiente código importa la hoja de estilos base en la ce tienda.css:

@import url(base.css);

Si tenemos distintas hojas de estilos a importar estas se ponen una debajo de la otra. El uso de múltiples reglas @import en un archivo CSS tiene el mismo impacto que añadir múltiples elementos de enlace al archivo HTML. Una de las ventajas de la regla @import es que simplifica su código HTML al colocar la decisión sobre qué hojas de estilo incluir y excluir en el archivo CSS en lugar de en el archivo HTML.
